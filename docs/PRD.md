## 1. 제품 개요

### 문제 정의

모임 주최자(수영, 헬스, 친구 모임 등)는 카카오톡 단톡방에서 공지, 참여 확인, 카풀 조율, 비용 정산을 모두 처리한다. 이 과정에서 메시지가 뒤섞이고 누락이 발생해 주최자의 관리 부담이 크다.

### 제품 목표

주최자 중심으로 한 곳에서 이벤트를 생성하고 참여자를 관리할 수 있는 최소 기능 웹앱. 1~2주 내 배포 가능한 MVP.

### 타겟 사용자

| 역할            | 설명                                      |
| --------------- | ----------------------------------------- |
| 주최자 (Host)   | 이벤트를 만들고 참여자를 관리하는 사용자  |
| 참여자 (Member) | 링크를 받아 이벤트에 참여 신청하는 사용자 |

### 접근 방식

- 이벤트 URL을 카카오톡/문자로 직접 공유
- Google OAuth 로그인 후 참여 신청
- 별도 앱 설치 불필요 (웹 브라우저로 접근)

## 2. 사용자 스토리

### 주최자 스토리

```
[이벤트 생성]
주최자가 Google 로그인
  → 대시보드(내 이벤트 목록) 진입
  → "새 이벤트 만들기" 클릭
  → 이벤트 생성 페이지에서 정보 입력 (제목/날짜/장소/인원/설명)
  → 생성 완료 → 이벤트 상세 페이지 이동
  → 공유 URL 복사 → 카카오톡/문자로 배포

[참여자 관리]
주최자가 이벤트 상세 → "참여자 관리" 탭 클릭
  → 참여자 목록(pending 상태) 확인
  → 개별 참여자 승인(confirmed) 또는 거절(rejected) 처리

[카풀 조율]
주최자가 이벤트 상세 → "카풀" 탭 클릭
  → 운전자/탑승자 신청 목록 확인
  → 수동 매칭 또는 자동 매칭 버튼 클릭
  → 매칭 결과 확인 및 수정

[정산 입력]
주최자가 이벤트 상세 → "정산" 탭 클릭
  → 총 비용 및 계좌 정보 입력
  → 1인당 금액 자동 계산 확인
  → 저장 → 참여자에게 공유
```

### 참여자 스토리

```
[이벤트 참여]
참여자가 카카오톡에서 이벤트 URL 수신
  → 브라우저에서 이벤트 상세 페이지 진입
  → Google 로그인 (미로그인 시 로그인 페이지로 리다이렉트)
  → 이벤트 상세 확인 (제목/날짜/장소/현재 인원)
  → "참여 신청" 버튼 클릭 → 상태: pending

[카풀 신청]
참여자가 이벤트 상세 → "카풀" 탭 클릭
  → 역할 선택: 운전 가능 | 탑승 필요
  → 인원/메모 입력 후 신청
  → 매칭 결과 대기 (새로고침으로 확인)

[정산 확인]
참여자가 이벤트 상세 → "정산" 탭 클릭
  → 1인당 정산 금액 확인
  → 주최자 계좌 정보 확인
  → 이체 완료
```

## 3. 핵심 기능 명세

### 3-1. 인증 (F010)

| 항목             | 내용                                                     |
| ---------------- | -------------------------------------------------------- |
| 기능             | Google OAuth 로그인/로그아웃                             |
| 구현 방식        | Supabase Auth (기존 구현 유지)                           |
| 비로그인 접근    | 이벤트 상세 페이지는 열람 가능, 참여 신청 시 로그인 요구 |
| 프로필 자동 생성 | 로그인 시 `profiles` 테이블에 자동 upsert                |

### 3-2. 이벤트 생성 및 공유 (F001)

| 항목          | 내용                                                                 |
| ------------- | -------------------------------------------------------------------- |
| 진입          | 대시보드 → "새 이벤트 만들기" 버튼                                   |
| 입력 필드     | 제목(필수), 날짜/시간(필수), 장소(필수), 최대 인원(필수), 설명(선택) |
| 공유 방법     | `/protected/events/[id]` URL 복사 버튼 제공                          |
| Server Action | `createEventAction`                                                  |
| 완료 후 이동  | 생성된 이벤트 상세 페이지                                            |

### 3-3. 이벤트 목록 및 상세 조회 (F002)

| 항목           | 내용                                                                 |
| -------------- | -------------------------------------------------------------------- |
| 대시보드       | 내가 주최한 이벤트 + 내가 참여 신청한 이벤트 목록                    |
| 상세 페이지    | 비로그인 사용자도 열람 가능                                          |
| 상세 표시 항목 | 제목, 날짜/시간, 장소, 설명, 현재 확정 인원 / 최대 인원, 주최자 이름 |
| 주최자 분기    | 이벤트 상세에서 주최자와 참여자에게 다른 UI 표시                     |

### 3-4. 참여 신청 및 관리 (F003)

**참여자 흐름**

| 상태      | 설명                     |
| --------- | ------------------------ |
| pending   | 참여 신청 직후 초기 상태 |
| confirmed | 주최자 승인 완료         |
| rejected  | 주최자 거절 처리         |

- 참여 신청: 이벤트 상세에서 "참여 신청" 버튼 (로그인 필요)
- 최대 인원 초과 시 신청 불가 (confirmed 기준 체크)
- 중복 신청 방지

**주최자 흐름**

- 참여자 관리 페이지: 신청자 목록 테이블 표시
- 각 행에 "승인" / "거절" 버튼
- Server Action: `updateMemberStatusAction`

### 3-5. 카풀 신청 및 매칭 (F004)

**신청 (참여자/주최자 공통)**

| 필드  | 설명                                           |
| ----- | ---------------------------------------------- |
| role  | `driver` (운전 가능) / `passenger` (탑승 필요) |
| seats | 탑승 가능 인원 (driver만 입력)                 |
| memo  | 출발지, 경유지 등 메모                         |

**매칭 (주최자 전용)**

- 카풀 페이지에서 driver-passenger 매칭
- `matchCarpoolAction`으로 수동 매칭 처리
- 매칭 결과 테이블 표시 (운전자 → 탑승자 목록)

### 3-6. 정산 (F005)

| 항목          | 내용                                               |
| ------------- | -------------------------------------------------- |
| 입력          | 주최자: 총 비용, 계좌 정보(은행명/계좌번호/예금주) |
| 계산 방식     | `total_cost / confirmed 참여자 수`                 |
| 표시          | 참여자: 1인당 금액 + 계좌 정보                     |
| Server Action | `createSettlementAction`                           |
| 결제 연동     | MVP 제외 (계좌 메모로 대체)                        |

## 4. 기술 스택

### 프론트엔드

| 기술         | 버전    | 용도                                        |
| ------------ | ------- | ------------------------------------------- |
| Next.js      | 15.5.3  | App Router, Server Actions                  |
| React        | 19      | UI 라이브러리                               |
| TypeScript   | 5.x     | 타입 안전성                                 |
| Tailwind CSS | 3.4.x   | 스타일링                                    |
| shadcn/ui    | latest  | UI 컴포넌트 (Button, Card, Table, Badge 등) |
| Lucide React | 0.511.x | 아이콘                                      |

### 백엔드

| 기술                   | 용도               |
| ---------------------- | ------------------ |
| Supabase Auth          | Google OAuth 인증  |
| Supabase PostgreSQL    | 데이터베이스       |
| Supabase RLS           | 행 수준 보안 정책  |
| Next.js Server Actions | 폼 처리 및 DB 연동 |

### 배포

| 기술           | 용도                |
| -------------- | ------------------- |
| Vercel         | Next.js 최적화 배포 |
| Supabase Cloud | DB 및 Auth 호스팅   |

### DB 스키마 (핵심 필드)

**`profiles`**

- `id` → auth.users.id
- `email`, `full_name`, `avatar_url`
- `created_at`

**`events`**

- `id` (UUID)
- `host_id` → profiles.id
- `title`, `description`, `location`
- `event_date`, `max_members`
- `created_at`

**`event_members`**

- `id` (UUID)
- `event_id` → events.id
- `user_id` → profiles.id
- `status` (`pending` / `confirmed` / `rejected`)
- `created_at`

**`carpools`**

- `id` (UUID)
- `event_id` → events.id
- `user_id` → profiles.id
- `role` (`driver` / `passenger`)
- `seats`, `memo`, `matched_driver_id`

**`settlements`**

- `id` (UUID)
- `event_id` → events.id
- `total_cost`
- `account_info` (계좌 정보 JSON)
- `created_at`

### RLS 정책 요약

| 테이블        | 조회             | 수정/삭제        |
| ------------- | ---------------- | ---------------- |
| events        | 전체 공개        | 주최자만         |
| event_members | 본인 또는 주최자 | 본인 또는 주최자 |
| carpools      | 본인 또는 주최자 | 본인 또는 주최자 |
| settlements   | 참여자 + 주최자  | 주최자만         |

### 페이지 구조

```
app/
├── page.tsx                          → /protected/events 리다이렉트
├── auth/
│   ├── login/page.tsx                → Google 로그인
│   ├── sign-up/page.tsx              → 회원가입 (OAuth 방식)
│   └── callback/route.ts            → OAuth 콜백 처리
└── protected/
    ├── layout.tsx                    → 공통 네비게이션
    ├── page.tsx                      → 대시보드 (내 이벤트 목록)
    └── events/
        ├── new/page.tsx              → 이벤트 생성
        └── [id]/
            ├── page.tsx              → 이벤트 상세 (주최자/참여자 분기)
            ├── members/page.tsx      → 참여자 관리 (주최자 전용)
            ├── carpool/page.tsx      → 카풀 신청/확인
            └── settlement/page.tsx   → 정산
```

## 5. 성공 기준

### MVP 완료 조건

| 기준                                                         | 검증 방법                            |
| ------------------------------------------------------------ | ------------------------------------ |
| 주최자가 이벤트를 생성하고 URL을 공유할 수 있다              | 생성 후 URL 복사 동작 확인           |
| 참여자가 URL로 접근하여 Google 로그인 후 참여 신청할 수 있다 | 비로그인 → 로그인 → 신청 플로우 확인 |
| 주최자가 pending 신청을 승인/거절할 수 있다                  | 상태 변경 후 DB 반영 확인            |
| 카풀 신청 및 매칭 결과를 확인할 수 있다                      | 드라이버-패신저 매칭 테이블 표시     |
| 정산 금액이 자동 계산되어 참여자에게 표시된다                | 총비용 입력 → 1인당 금액 표시        |
| RLS 정책으로 권한 외 데이터에 접근이 차단된다                | 타 사용자 데이터 직접 조회 시도      |

### 배포 조건

- Vercel에서 빌드 오류 없음
- `npm run build` 로컬 통과
- 모바일 브라우저(Chrome/Safari)에서 주요 기능 동작

## 7. 향후 고려사항

MVP 이후 필요 시 추가할 기능 목록 (현재 범위 외):

| 기능                            | 이유                                   |
| ------------------------------- | -------------------------------------- |
| 실시간 알림 (Supabase Realtime) | 참여 신청 시 주최자에게 즉시 알림 필요 |
| 카카오 소셜 로그인              | 타겟 사용자가 카카오 사용자와 겹침     |
| 이벤트 이미지 업로드            | 장소 사진, 안내 이미지 첨부 필요 시    |
| 카풀 자동 매칭 알고리즘         | 인원이 많아질 경우 수동 매칭 비효율    |
| 정산 상태 추적                  | 참여자별 이체 완료 여부 체크 기능      |
| 이벤트 취소/수정 알림           | 일정 변경 시 참여자 일괄 통보          |
| 결제 연동 (토스페이먼츠 등)     | 앱 내 정산 완결성 확보                 |
| 반복 이벤트                     | 정기 모임 (매주 수요일 등) 자동 생성   |
| 이벤트 댓글/Q&A                 | 참여자 문의 및 공지 기능               |
| 관리자 대시보드                 | 다수 이벤트를 운영하는 주최자용 통계   |
